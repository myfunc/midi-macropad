# Шпаргалка по аудио-маршрутизации

Краткий справочник по настройке аудио: Voicemeeter Banana, OBS, многодорожечная запись.

---

## 1. Маршрутизация Voicemeeter Banana

```
                        +------------------+
  Blue Yeti (USB) ────> | Strip[0] "MIC"   | ──────────────────> B1 (виртуальный микрофон)
                        +------------------+

                        +------------------+
  VM Input  ──────────> | Strip[3] "DESKTOP"| ─────────────────> A1 (наушники)
  (системный звук)      +------------------+

                        +------------------+
  VM AUX Input ───────> | Strip[4] "SEND2MIC"| ────────────────> A1 (наушники)
  (доп. аудио с ПК)     +------------------+    (toggle B1 при необходимости)


  +---------------------+
  | Bus A1 "HEADPHONES" | = Intel SST наушники (аппаратный выход, monitor ON)
  +---------------------+

  +---------------------+
  | Bus B1 "MIC B1"     | = Виртуальный микрофон для Discord/OBS/etc (EQ ON)
  +---------------------+
```

### Что куда идёт

| Strip / Bus | Устройство | Куда направлено | Назначение |
|---|---|---|---|
| Strip[0] "MIC" | Blue Yeti mic | B1 only | Микрофон для Discord, OBS и т.д. |
| Strip[3] "DESKTOP" | VM Input | A1 only | Системный звук в наушники |
| Strip[4] "SEND2MIC" | VM AUX Input | A1 only | Вброс звука ПК в микрофонный канал (B1 по необходимости) |
| Bus A1 | Intel SST наушники | Аппаратный выход | Мониторинг, monitor ON |
| Bus B1 | Виртуальный выход | Discord/OBS/etc | EQ включён |

## 2. Источники аудио в OBS и связь с Voicemeeter

| Источник в OBS | Тип захвата | Что захватывает |
|---|---|---|
| **Desktop Audio** | WASAPI Output Capture | Системный звук (всё, что идёт на A1 / динамики) |
| **Mic/Aux** | WASAPI Input Capture | Voicemeeter B1 виртуальный выход (микрофон после обработки VM) |
| **AUX Input** (опционально) | WASAPI Input Capture | Voicemeeter B2 или другой виртуальный выход |

---

## 3. Независимое управление каналами

Каждый канал управляется отдельно через OBS WebSocket API (SetInputVolume):

| Канал | OBS-источник | API-вызов | Управление с Midi Macropad |
|---|---|---|---|
| Десктоп | "Desktop Audio" | `SetInputVolume` | Да, отдельный пэд |
| Микрофон | "Mic/Aux" | `SetInputVolume` | Да, отдельный пэд |
| AUX | "AUX Input" | `SetInputVolume` | Да (если настроен) |

Каждый канал можно мьютить независимо с пэдов Midi Macropad.

## 4. Многодорожечная запись OBS

| Дорожка | Содержимое | Назначение |
|---|---|---|
| Track 1 | Все источники (микс) | Для стриминга |
| Track 2 | Desktop only | Системный звук отдельно |
| Track 3 | Mic only | Микрофон отдельно |
| Track 4 | AUX only | Дополнительный звук отдельно |

### Как настроить

1. Открыть OBS: **Edit** -> **Advanced Audio Properties**
2. Для каждого источника отметить галочками нужные дорожки
3. Формат записи: **MKV** (рекомендуется -- поддерживает несколько дорожек, устойчив к вылетам)

---

## 5. Стерео

- OBS: **Settings** -> **Audio** -> **Channels** = **Stereo**
- Voicemeeter B1 уже выдаёт стерео-сигнал
- Каждая аудиодорожка наследует стерео-настройку

## 7. Настройки EQ на Voicemeeter B1 (справка)

EQ применяется к виртуальному микрофонному выходу (Bus B1).

| Полоса | Частота | Усиление | Назначение |
|---|---|---|---|
| 1 | 50 Hz | -3.0 dB | Подавление гула / низкочастотного шума |
| 2 | 200 Hz | -1.0 dB | Убираем "грязь" в нижней середине |
| 3 | 800 Hz | 0.0 dB | Нейтральная середина (без изменений) |
| 4 | 2000 Hz | +1.0 dB | Присутствие, разборчивость голоса |
| 5 | 8000 Hz | +1.0 dB | Воздух / ясность |
| 6 | 12000 Hz | +0.5 dB | Блеск / лёгкое усиление верхов |
